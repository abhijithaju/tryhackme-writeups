Summary
Covered the basics of navigating and interacting with a Linux system 
through the terminal. All tasks were done on a remote Linux machine 
deployed through TryHackMe.

Commands covered

pwd - prints current working directory
ls - lists files and folders in current location
cd - moves into a specified directory
cat - reads and displays contents of a file
find - searches for files across the system
grep - searches for a specific string inside a file

Operators covered

> - redirects output into a file, replacing existing content
>> - appends output to a file without removing existing content
& - runs a command in the background

Concepts learned
- Linux terminal is the primary interface for cybersecurity work
- Files and directories are navigated using pwd, ls, and cd
- cat reads files, cd does not - cd only works on directories
- grep can search log files for specific strings, useful in 
  investigation and forensics
- The difference between overwrite and append operators
